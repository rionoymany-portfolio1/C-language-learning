# C Programming: Systems Thinking & Security Foundation

> Age: 14 | Self-directed learner 
> Building foundational systems knowledge toward security research and financial technology. (2026)

---

## The Dual-Core Objective

1. **Systems & Security Foundation:** Mastering memory management, pointers, and low-level C syntax to build a deep understanding of how software works at the hardware level — including how vulnerabilities like buffer overflows arise and why they matter.

2. **Business Intelligence:** Applying programming logic to financial problems, risk modeling, and understanding the economic impact of software systems.

---

## Curriculum Roadmap (4-Week Intensive)

### Week 1: Core Fundamentals & Logic ✅
- **Focus:** Syntax, basic math, decision-making (If-Else)
- **Systems Thinking:** Understanding program flow and how a CPU executes decisions
- **Business Insight:** Building simple decision-support logic for financial conditions
- **Projects:** `hello.c`, `calculator.c`, `even_odd.c`, `grade_check.c`, `max_number.c`

### Week 2: Iteration & Automation ✅
- **Focus:** Nested If, Switch Case, Loops (While, For, Do-While)
- **Systems Thinking:** Analyzing how iterative patterns behave under different input conditions — a foundation for understanding automated probing logic
- **Business Insight:** Automating batch financial calculations and inventory processing
- **Projects:** `atm_simulation.c`, `brute_force_sim.c`, `multi_tax_calc.c`, `multiplication_matrix.c`

### Week 3: Data Management & Memory Patterns ✅
- **Focus:** getchar/putchar, gets/puts, 1D & 2D Arrays
- **Systems Thinking:** Understanding how data occupies contiguous memory blocks — a prerequisite for studying memory safety and how corruption vulnerabilities occur
- **Business Insight:** Multi-quarter financial analysis and data matrix reporting
- **Key Achievement:** Implemented 2D arrays for complex data structures with statistical averaging

### Week 4: Modular Programming & Function Scope 🔜
- **Focus:** Functions, parameters, return values, scope
- **Goal:** Designing clean, reusable code and understanding the call stack
